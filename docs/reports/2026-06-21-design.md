## Executive Summary

This inaugural design research run establishes **docs/DESIGN-IDENTITY.md** as the single source of truth for rankings123.com's visual identity and files **6 concrete design tickets** to push the site from a clean, functional Apple Sports-inspired base toward a **distinctive, memorable premium sports rankings hub**.

**Key insight:** Rankings123 has a solid foundation (token-based theming, per-sport accents, 3 design variants, clean Apple Sports aesthetic), but lacks **signature moments** — a distinctive logo, cohesive icon system, refined interaction states, and visual depth that would make it screenshot-worthy and award-competitive.

**Filed tickets:**
1. `logo-wordmark` — distinctive logo/wordmark (priority 1)
2. `button-state-system` — full 6-state button system per Clay 2026 (priority 2)
3. `hero-depth-treatment` — 3D depth/layered gradients for hero sections (priority 2)
4. `loading-state-system` — branded loading skeletons (priority 3)
5. `icon-system` — cohesive sport/UI icon set (priority 3)
6. `empty-state-design` — friendly empty states (priority 3)

All ladder up to the **DESIGN-IDENTITY.md** north star: "Live data, living design" — a premium sports editorial that feels animated without being gimmicky.

## Research Sources (This Run)

### Primary: Clay Blog (clay.global/blog)
Reviewed 3 recent articles, logged in DESIGN-IDENTITY.md:

   - **Takeaway:** Full 6-state system (enabled, hover, focus, pressed, disabled, loading) with minimum 44×44px touch targets, visible WCAG 2.2 focus rings, and specific action labels ("Save changes" not "Submit").
   - **Application:** Current nav tabs/toggles need refinement — filed ticket `button-state-system`.

   - **Takeaway:** Use selectively (modals, floating cards), requires colorful backgrounds, limit to 2–3 blur layers (performance), dark mode needs higher opacity (20–30%).
   - **Application:** Current `backdrop-blur` on nav is appropriate; noted for future modal/overlay design (no immediate ticket).

   - **Takeaway:** Colorful + minimal + clean + 3D/animation. High scores (8–9.6) for balancing vibrant visuals with functional clarity. Demonstrates that sports data sites can be award-worthy without sacrificing usability.
   - **Application:** Inspired ticket `hero-depth-treatment` (subtle 3D depth for hero/cards).

### Secondary: Godly, Awwwards (sports/data viz trends)
- **Trend:** High-contrast scores dominate best sports sites; oversized displays, bold team color accents, mobile-first layouts.
- **Trend:** Data viz sites that win awards balance **colorful + minimal + clean** (not one extreme).
- **Validation:** Rankings123's per-sport accent system (ATP lime, WTA magenta, World Cup green) aligns with 2026 sports design best practices.

## Current Site Audit

### Strengths ✅
- **Token-based design system:** All colors via CSS variables (`--bg`, `--surface`, `--accent`, etc.) — easy to theme, accessible, professional.
- **Per-sport accent system:** Dynamic accent colors via `data-sport` attribute (ATP lime, WTA magenta, etc.) — distinctive and flexible.
- **3 design variants:** Court (default premium), Broadcast (TV scoreboard), Classic (editorial paper) — shows design ambition and experimentation.
- **Typography:** Archivo display font is a good choice (sporty, bold, not generic Inter/Roboto). Tabular-nums enforced for data.
- **Motion:** Pulsing live-dot is a **signature element** (recognizable, well-executed). Count-up number animations on refresh are smooth.
- **Data-first:** Rankings/scores are always prominent, design doesn't obscure content (CX-first principle enforced).
- **Responsive:** Mobile-friendly cards, sticky nav, generous touch targets.

### Gaps / Opportunities 🔧
1. **Logo/branding:** Text-only "RANKINGS 123" with live dot — lacks a distinctive wordmark/logo that would be memorable on social, favicon, or as a standalone mark. *Ticket: `logo-wordmark`*
2. **Button/interaction states:** Nav tabs have hover/active, but missing visible focus rings, pressed state feedback, and full 6-state coverage across all controls. *Ticket: `button-state-system`*
3. **Visual depth:** Hero and cards are flat gradients — could benefit from subtle 3D layering (stacked washes, transform-based depth) inspired by Awwwards nominees. *Ticket: `hero-depth-treatment`*
4. **Loading states:** No branded skeleton loaders or spinners — data refresh is invisible until content pops in (jarring). *Ticket: `loading-state-system`*
5. **Icons:** Heavy reliance on emoji (🏆🎾🚴) — works but not cohesive or premium. Replace with designed SVG icon system. *Ticket: `icon-system`*
6. **Empty states:** No designed treatment for "no search results", "no active tournaments", filters returning zero — harsh UX when data is missing. *Ticket: `empty-state-design`*
7. **Favicon:** Still default Next.js favicon (already tracked in existing ticket `favicon` — good, didn't duplicate).
8. **OG images:** Generic/missing social preview cards — future ticket (not filed this run, noted in DESIGN-IDENTITY.md roadmap).

## DESIGN-IDENTITY.md (North Star Established)

Created **docs/DESIGN-IDENTITY.md** as the living spec for rankings123's visual identity. Key sections:

### Signature Idea
**"Live data, living design."** The site feels animated without being gimmicky — pulsing indicators, smooth transitions, vibrant per-sport accents that shift as you navigate. Premium sports editorial meets real-time dashboard.

### Type System
- **Body:** Geist Sans (neutral, readable)
- **Display:** Archivo 600–900 (sporty, bold, distinctive)
- **Monospace:** Geist Mono (tabular data)
- **Next evolution:** More dramatic scale contrast (whisper-to-shout typographic rhythm).

### Color & Accent Logic
- **Dark (default):** Near-black `#0a0b0f`, high contrast, premium feel
- **Light:** Soft off-white `#f6f7f9`, crisp editorial
- **Per-sport accents:** ATP lime `#b6f23c`, WTA magenta `#f472b6`, World Cup green `#22c55e`, Cycling amber `#fbbf24`
- **Semantic:** Up/down (green/red), trophy/podium (gold/silver/bronze)

### Motion Principles
- **Active:** Pulse dot (1.6s loop), count-up numbers, hover transitions (150–200ms)
- **Constraints:** GPU-cheap only (`transform`/`opacity`), respect `prefers-reduced-motion`
- **Next evolution:** Cohesive motion language (loading skeletons, page transitions, refresh cues).

### Component Language
- Documented existing patterns: HeroBanner, SportCard, RankBadge, Movement chips, LiveDot
- Identified gaps: Logo, loading states, empty states, icon system, OG images, button refinement

### Do's & Don'ts
- ✅ Use design tokens exclusively, let data dominate, test dark+light+all variants, WCAG AA contrast, low ad density, mobile-first, preserve CWV
- ❌ Raw Tailwind grays, purposeless motion, obscure data, new fonts, one-off spacing, placeholder UI, excessive `backdrop-blur`, ignore `prefers-reduced-motion`

### Evolution Roadmap
- **Phase 1 (Foundation):** Per-sport accents ✅, dark/light ✅, token-based CSS ✅, Apple Sports base ✅, **logo/favicon** (tickets filed)
- **Phase 2 (Refinement):** Button states, hero depth, type scale, icon system, loading/empty states (tickets filed this run)
- **Phase 3 (Signature Moments):** OG images, micro-interactions, data storytelling, ambient motion (future)

**New tickets (all tagged `design`, parent `rankings123`):**

1. **`logo-wordmark`** (priority 1, tags: design, brand, seo)
   - Distinctive SVG logo/wordmark replacing text-only "RANKINGS 123"
   - Incorporates live-dot motif, works across dark/light + all sizes (16px favicon to hero)
   - Acceptance: SVG in src/assets/, integrated in Nav, legible at all sizes, no logo-generator artifacts

2. **`button-state-system`** (priority 2, tags: design, ux, a11y)
   - Full 6-state button system per Clay 2026: enabled, hover, focus, pressed, disabled, loading
   - WCAG 2.2 visible focus rings, 44×44px touch targets, smooth transitions
   - Apply to: nav tabs, theme toggle, design switcher, filters, future CTAs
   - Acceptance: All 6 states designed, focus rings compliant, mobile touch targets ≥44px, works across themes

3. **`hero-depth-treatment`** (priority 2, tags: design, ui, engagement)
   - Subtle 3D depth for HeroBanner and SportCards: layered gradients, CSS transform parallax/depth
   - Inspired by Awwwards World Cup 2026 (colorful + 3D + clean)
   - Acceptance: Enhanced gradients (2–3 layers max), subtle 3D hover/scroll effects, no CWV regression, GPU-efficient, respects `prefers-reduced-motion`

4. **`loading-state-system`** (priority 3, tags: design, ux)
   - Branded skeleton screens + spinners using per-sport accent colors
   - Smooth fade-in transitions, realistic layout previews
   - Apply to: LiveRankingTable, AtpDeepRankingTable, WorldCupTable, SportCards
   - Acceptance: Skeletons for tables/cards/hero, accent-colored pulse, respects `prefers-reduced-motion`, works across themes

5. **`icon-system`** (priority 3, tags: design, ui, brand)
   - Cohesive SVG icon set replacing emoji (🏆🎾🚴)
   - Style: bold, geometric, consistent stroke weight (2–2.5px), rounded caps
   - Icons for: tennis, football, cycling, trophy, live, arrows, theme toggle, design switcher
   - Acceptance: SVG icons created, consistent style, accessible (aria-label), legible dark+light, applied across Nav/HeroBanner/SportCard

6. **`empty-state-design`** (priority 3, tags: design, ux)
   - Friendly empty states for: no search results, no active tournaments, zero filter matches
   - Uses per-sport accent colors, clean messaging + optional CTA
   - Acceptance: Empty state components, icon/illustration + message + CTA, premium aesthetic, friendly copy, works across themes

## Recommended Priority for Planner

**Build order:**
1. **`logo-wordmark`** (highest impact, blocks favicon completion, SEO/brand value)
2. **`button-state-system`** (a11y + UX debt, affects all interactive controls)
